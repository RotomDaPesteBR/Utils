# Result Pattern Core Types

Detailed documentation for the fundamental types of the Result Pattern.

## Types

*   **[Result](Result.md)**: Base class for success/failure results without a return value.
*   **[Result<TValue>](Result.md)**: Generic result class encapsulating a success value.
*   **[Success](Success.md)**: Metadata for successful outcomes.
*   **[Error](Error.md)**: Standardized error representation.
*   **[ErrorDetail](ErrorDetail.md)**: Contextual information for errors.
*   **[TaskResult](TaskResult.md)**: Awaitable wrapper for asynchronous results.

## Sub-directories

*   **[Errors](Errors/README.md)**: Catalog of specialized error modules.
*   **[Extensions](Extensions/README.md)**: Fluent API extension methods.
*   **[Exceptions](Exceptions/README.md)**: Specialized exceptions for the library.
